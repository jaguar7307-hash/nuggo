# 미리보기 화면 - 전화/카톡/포트폴리오 기능 가이드

## ✅ 구현된 기능

| 기능 | 구현 상태 | 설명 |
|------|----------|------|
| **전화 (PHONE)** | ✅ 구현됨 | `tel:` URI로 전화 앱 실행 |
| **문자 (MESSAGE)** | ✅ 구현됨 | `sms:` URI로 문자 앱 실행 |
| **카카오톡 (KAKAO)** | ✅ 구현됨 | 오픈채팅 링크 또는 `open.kakao.com/me/ID` |
| **포트폴리오** | ✅ 구현됨 | URL 열기 또는 파일 공유 |

---

## ❌ 안 되는 이유 (시뮬레이터 / 맥)

### 1. 전화 (PHONE) - 안 되는 이유

| 환경 | 이유 |
|------|------|
| **iOS 시뮬레이터** | 시뮬레이터에는 **전화 앱이 없음**. `tel:` URL은 실제 아이폰에서만 동작 |
| **macOS 앱** | 맥에는 전화 앱이 없음 (FaceTime은 별도) |
| **에디터에서 전화번호 미입력** | `data.phone`이 비어 있으면 아무 동작 안 함 → "전화번호를 입력해 주세요" 표시 |

**해결:** 실제 아이폰에 앱 설치 후 테스트

---

### 2. 카카오톡 (KAKAO) - 안 되는 이유

| 환경 | 이유 |
|------|------|
| **iOS 시뮬레이터** | 시뮬레이터에 **카카오톡 앱이 설치되어 있지 않음** |
| **macOS** | 맥에 카카오톡이 없으면 `open.kakao.com` 링크만 브라우저로 열림 |
| **에디터에서 카카오 ID 미입력** | `data.kakao`가 비어 있으면 → "카카오톡 오픈채팅 ID 또는 링크를 입력해 주세요" 표시 |

**해결:** 실제 아이폰 + 카카오톡 설치 후 테스트. 또는 `https://open.kakao.com/me/아이디` 형식 링크 입력 시 브라우저로 열림

---

### 3. 포트폴리오 - 안 되는 이유

| 상황 | 이유 |
|------|------|
| **링크/파일 둘 다 없음** | 에디터에서 포트폴리오 URL 또는 파일을 입력하지 않음 → "포트폴리오 링크 또는 파일을 입력해주세요" 표시 |
| **파일 공유 (시뮬레이터)** | `share_plus`로 PDF 공유 시, 시뮬레이터에서는 공유 시트가 제한적일 수 있음 |
| **URL 형식 오류** | `http://` 또는 `https://` 없이 입력 시 자동 보정되지만, 잘못된 URL은 열리지 않음 |

**해결:** 에디터에서 포트폴리오 링크(예: https://portfolio.com) 또는 PDF 파일 첨부

---

## 📋 체크리스트 (기능이 동작하려면)

1. **에디터에서 입력 확인**
   - 전화번호 (연락처 > 전화)
   - 카카오톡 ID 또는 오픈채팅 링크
   - 포트폴리오 URL 또는 파일 첨부

2. **실제 기기에서 테스트**
   - 시뮬레이터 ❌ → 실제 아이폰 ✅
   - 전화/문자/카톡은 **실제 기기**에서만 정상 동작

3. **저장 후 미리보기**
   - 에디터에서 저장 버튼 클릭 후 미리보기 아이콘 탭

---

## 🔧 개선 사항 (적용됨)

- 빈 값일 때 안내 SnackBar 메시지 추가
- `launchUrl` 실패 시 환경별 안내 메시지 (시뮬레이터/맥)
- iOS Info.plist에 `LSApplicationQueriesSchemes` (tel, sms, mailto, https) 추가
