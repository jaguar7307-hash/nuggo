# 명함 페이지 레이아웃 구조 비교 (React 원본 vs Flutter)

### React 원본 구조
```
┌─────────────────────────────────────────┐
│  [Guide] (우측 상단 절대)                 │
│  "실시간 미리보기" (작은 라벨)             │
│  ┌──────────┐     ┌──┐                   │
│  │          │     │Send│ (주황 tangerine)  │
│  │  명함카드  │     ├──┤                   │
│  │ 220×352  │     │Share│ (주황 50%)      │
│  │ scale 95%│     ├──┤                   │
│  │          │     │ QR │ (흰색)            │
│  │  [👁미리보기]  │  (카드 하단 중앙, 절반 겹침) │
│  └──────────┘     └──┘                   │
├─────────────────────────────────────────┤
│  배경 테마 (탭: professional/personal/creative) │
│  [썸네일] [썸네일] [AI Gen] [Add]          │
├─────────────────────────────────────────┤
│  프로필 목록 | [폰트]  [새 프로필] [프로필1] [프로필2]... │
├─────────────────────────────────────────┤
│  1. 한줄 소개 [AI 자동생성]               │
│  2. 기본 정보 (이름, 직함, 회사)          │
│  3. 사진 및 로고                          │
│  4. 연락처 (전화, 문자, 이메일, 카카오)    │
│  5. 링크 및 SNS                           │
│  6. 주소                                  │
│  [프로필 저장 / 업데이트]                  │
└─────────────────────────────────────────┘
```

### Flutter 현재 구조 (차이점)
| 항목 | React 원본 | Flutter 현재 |
|------|------------|--------------|
| Guide 버튼 | 우측 상단 있음 | **없음** |
| 미리보기 버튼 | 카드 **하단 중앙에 절반 겹침** (카드 일부) | 카드 **아래 별도** ElevatedButton |
| 우측 액션 | Send(주황), Share(주황 50%), QR(흰색) | secondary, secondary 0.5, grey |
| 카드 크기 | 220×352 고정, scale 0.95 | 220 + AspectRatio, scale 0.95 (비슷) |

## 2. 미리보기 화면 (PreviewView / PreviewScreen)

### React 원본 구조 (풀스크린 명함 뷰)
```
┌─────────────────────────────────────────┐
│ [상대방에게 보이는 화면] (좌상단, 3초 후 사라짐)  [X 닫기] (우상단)
│                                          │
│            NUG GO (로고)                  │
│            ───────                       │
│                                          │
│            "슬로건"                       │
│       (프로필 이미지 있으면 32×32 원형)     │
│                                          │
│            이름 (text-5xl 큼)             │
│            직함 (골드 #d4b98c)            │
│            회사명                         │
│                                          │
│  [📞]    [💬]    [✉️]                    │
│  Phone   Message  Email                  │
│  [🌐]    [💬]    [공유]                   │
│  Website  Kakao   SNS                    │
│                                          │
│        📍 주소 (클릭 시 지도)              │
│                          [👤 연락처 저장] │
├─────────────────────────────────────────┤
│  "나의 모든 것을 탑카드 하나로!"           │
│  [  나도 명함 만들기  ] (풀폭 버튼)        │
└─────────────────────────────────────────┘
```
- **배경**: 명함 테마(이미지/색상)가 **전체 화면**.
- **콘텐츠**: 카드가 아닌 **풀스크린 레이아웃** (로고, 슬로건, 이름, 3×2 액션+라벨, 주소, 하단 CTA).

### Flutter 현재 구조 (차이점)
| 항목 | React 원본 | Flutter 현재 |
|------|------------|--------------|
| 배경 | 명함 테마 **풀스크린** | **회색 Scaffold** + 중앙에 카드만 |
| 로고 | NUG GO 상단 | 없음 |
| 슬로건/이름/직함 | 큰 타이포, 풀스크린 배치 | DigitalCard **한 장** 안에만 표시 |
| 액션 | **3×2 그리드 + 라벨** (Phone, Message, Email...) | 카드 내 3개 아이콘만 |
| 연락처 저장 | 우측 플로팅 버튼 | 없음 |
| 하단 CTA | "나도 명함 만들기" 풀폭 버튼 | 없음 |

## 3. 명함 카드 위젯 (DigitalCard)

### React 원본
- **비율**: aspect 1/1.6, rounded-2rem
- **배경**: hex → paper-fibers 패턴 + hex 색 / 이미지 → 이미지 + paper-fibers
- **구성**:
  - 상단: 프로필 이미지(24×24) **또는** 슬로건(구분선 + "...")
  - 중앙: 이름, 직함(작은 대문자), 회사명
  - 하단: **6개 액션** (call, mail, language, sms, forum, share) **3열 그리드** + 주소 pill

### Flutter 현재
- **비율/모서리**: 동일
- **배경**: paper-fibers **없음**
- **액션**: **3개만** (call, mail, language) → 6개 + 3열 그리드로 맞춰야 함

## 수정 방향 요약

1. **PreviewScreen**: 회색+카드 → **풀스크린 명함 배경** + 원본과 동일한 블록(로고, 슬로건, 이름, 3×2 액션+라벨, 주소, 연락처 저장, 하단 CTA).
2. **EditorScreen**: Guide 버튼 추가, 미리보기 버튼을 **카드 하단 중앙 겹침**, 우측 버튼 **tangerine/흰색** 적용.
3. **DigitalCard**: 액션 **6개 3열**, paper-fibers 텍스처는 가능 시 적용(또는 생략).
