# NUGGO — 백엔드 없이 가능한 기능 vs 백엔드 개발 후 가능한 기능

## ✅ 백엔드 없이 구현된 기능 (로컬/기기만 사용)

- **프로필·명함 CRUD**: 생성, 수정, 삭제, 다중 프로필 (로컬 `SharedPreferences` 저장)
- **설정**: 다크모드, 알림, 마케팅 수신, 앱 잠금(PIN/생체), 사운드/햅틱 등 (로컬 저장)
- **미리보기**: 에디터에서 편집한 명함을 그대로 풀스크린 미리보기
- **명함 공유**
  - **QR 코드**: 내 명함 하단 플로팅 버튼 → 선택한 명함의 `shareLink`(또는 기본 URL)를 QR로 표시 (다이얼로그)
  - **공유(Share)**: 내 명함 화면 공유 버튼, Preview/디지털 카드의 SNS 공유 → `share_plus`로 링크 공유
- **연락처 액션**: 전화, SMS, 이메일, 웹사이트/포트폴리오 링크 등 `url_launcher`로 기기 기본 앱 실행
- **게스트/로컬 계정**: 로컬에 사용자·세션 저장, 로그인/로그아웃 (실제 서버 인증 없음)
- **결제 모달**: UI만 (프로 구독은 로컬 플래그 변경으로 시뮬레이션)

---

## 🔶 백엔드 개발 이후 가능한 기능

### 1. **실제 명함 링크(shareLink) 동작**
- 현재: QR/공유 시 사용자가 입력한 URL(예: `nuggo.me/username`)을 그대로 표시만 함. 해당 URL에 접속해도 서버에 페이지가 없어 실제 명함 페이지가 열리지 않음.
- 백엔드 후: `shareLink`에 해당하는 **명함 전용 페이지**를 서버에서 제공 (예: 짧은 URL → 프로필/명함 뷰).

### 2. **조회수·발송수·인사이트**
- 현재: 내 명함의 Views/Sends, INSIGHT ANALYSIS(Total Views, Success Rate, 주간 차트)는 **목업(고정/랜덤)** 데이터.
- 백엔드 후: 명함 링크 방문·공유·스캔 시 서버에 이벤트 저장 → **실제 조회수/발송수** 집계 및 차트/인사이트 API 연동.

### 3. **최근 발송 기록(Recent Send History)**
- 현재: 고정 목업 리스트 (예: "David Chen, Sent via NFC Tag").
- 백엔드 후: NFC/공유/링크 발송 시 서버에 기록 → **실제 발송 이력** 조회 및 "VIEW ALL" 연동.

### 4. **실제 회원 인증·동기화**
- 현재: 로컬 사용자/세션만 저장. 기기 바꾸면 데이터 없음.
- 백엔드 후: **회원가입·로그인(이메일/소셜)** → 프로필·설정 **클라우드 동기화**, 다중 기기에서 동일 데이터.

### 5. **NFC 발송**
- 현재: NFC 버튼은 UI만 있음.
- 백엔드 후: NFC 태그에 **명함 URL 또는 식별자** 기록, 상대방이 스캔 시 서버에서 명함 로드 및 **발송 이벤트** 저장.

### 6. **실제 결제·프로 구독**
- 현재: 결제 모달에서 "업그레이드" 시 로컬에서 Pro 플래그만 변경.
- 백엔드 후: **결제 게이트웨이** 연동, 구독 상태를 서버에서 관리 및 기기별 동기화.

### 7. **기타**
- **푸시 알림**: 서버에서 발송 (예: 명함 조회 알림).
- **명함 템플릿/테마**: 서버에서 목록 제공·업데이트.
- **관리자/분석 대시보드**: 조회·발송·사용자 통계를 백엔드에서 집계해 웹 등으로 제공.
